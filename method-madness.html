Method Madness

Playing around with Ruby today, it dawned on me that if everything is an object, then a method is an object.

def foo                                    
  puts "foo"
end

m = method(:foo)

puts m.class                    # -> Method
puts m.class.superclass         # -> Object

So, a method is an object: it's an instance of class Method and a subclass of class Object.

Then I asked myself, if an object can have a singleton class, can a method?

Well, it can. Look at this:

class << m
  def bar 
    puts "bar"
  end 
end


puts m.methods.include?(:bar)   # -> true
m.method(:bar).call             # -> bar

That's very funky. Although, I am not sure if it has any practical use.
